# UX検定基礎 学習サイト 画面デザイン改善 修正方針

- 作成日: 2026-07-05
- 目的: 学習の妨げになっている表示不具合（要素の重なり等）を根絶し、全画面を「見やすくリッチな」デザインに引き上げる
- 対象読者: **本方針を実行する後続エージェント**（この文書だけで作業に着手できるよう、監査結果・根拠・制約を本文に埋め込んである）
- 根拠: 2026-07-05 に実施した UI 監査（`site/css/style.css` 633行・`site/js/app.js` 1,103行・`site/index.html` の全読、全6画面のレンダリングロジック追跡）

## 0. 実行エージェントへの前提指示（必読）

### 0.1 触ってよいファイル / 触ってはいけないファイル

| ファイル | 扱い |
|---|---|
| `site/css/style.css` | **主戦場**。全面的に改修してよい |
| `site/js/app.js` | ビュー関数内の HTML テンプレート・class 名の変更は可。**学習ロジック（STORE / startQuiz / finishQuiz / 採点・保存まわり）は変更しない** |
| `site/index.html` | ヘッダー構造・meta の追加変更は可 |
| `site/js/data.js` | **生成物。編集禁止** |
| `docs/*.md`, `docs/syllabus_keywords.json` | コンテンツ層。**本件では触らない**（デザイン改善にコンテンツ変更は不要） |
| `site/build.js` | 触らない |

### 0.2 絶対ルール

1. **依存ゼロを維持する。** npm パッケージ・CDN・外部フォント・アイコンフォントを導入しない。`site/index.html` をブラウザで開くだけで動く状態を守る。アイコンは現状どおり絵文字、またはインライン SVG で賄う。
2. `docs` を触らないので `data.js` は変わらないが、各フェーズ完了時に `node site/build.js` を実行し exit 0 を確認する（CI と同条件の確認）。
3. localStorage のキー（`uxkentei.v1`, `uxtheme`）とデータ構造を変えない。既存ユーザーの学習進捗を壊さないこと。
4. ハッシュルーティング（`#/read/1` 等）の URL を変えない。
5. アクセシビリティを後退させない: `skip-link`、`aria-*` 属性、`:focus-visible`、`prefers-reduced-motion` 対応は維持・強化のみ。
6. ダークモードは `html[data-theme="dark"]` のトークン差し替え方式を維持する。**新しい色を追加するときは必ずライト/ダーク両方のトークンを定義**し、コンポーネント側では生の色値を書かない。
7. 各フェーズ完了ごとに §6 の手動確認チェックリストを実施 → コミット、を1サイクルとする。

### 0.3 実装しないと決めたこと（提案しない）

- Web フォント読み込み（依存ゼロ方針と衝突。システムフォントスタックのまま磨く）
- CSS フレームワーク / ビルドツール導入
- 学習機能の追加・変更（本件はあくまで見た目とレイアウトの改善）

## 2. デザインシステムの整備（Phase A の成果物）

すべての画面改修に先立ち、`style.css` の `:root` を以下の方針でトークン化する。**値の最終調整は実装時の見た目確認で行ってよいが、「トークン経由でしか使わない」原則は守る。**

### 2.1 追加・整理するトークン

```css
:root {
  /* スペーシング（4pxベース） */
  --sp-1: 4px;  --sp-2: 8px;  --sp-3: 12px; --sp-4: 16px;
  --sp-5: 20px; --sp-6: 24px; --sp-7: 32px; --sp-8: 40px;

  /* タイポグラフィスケール（これ以外のfont-sizeを新規に書かない） */
  --fs-xs: 12px; --fs-sm: 13.5px; --fs-base: 15px; --fs-md: 16.5px;
  --fs-lg: 19px; --fs-xl: 24px;  --fs-2xl: 30px;  --fs-3xl: 38px;

  /* 高さ3段階のエレベーション */
  --shadow-1: /* 現 --shadow-sm 相当 */;
  --shadow-2: /* 現 --shadow 相当 */;
  --shadow-3: /* ホバー・モーダル級。新設 */;

  /* 文字用の濃色トークン（P1-4対策。ライト/ダーク両方定義） */
  --teal-text: …; --amber-text: …; --green-text: …; --red-text: …;

  /* カード標準パディング（P1-1対策） */
  --card-pad: 20px; --card-pad-lg: 24px;
}
```

- 既存の `--radius`(14px)/`--radius-sm`(10px) は維持し、`--radius-lg: 20px`（hero・フラッシュカード用）を追加。
- チップ類の文字色を `--amber` → `--amber-text` 等へ置換（背景 soft 系との組で 4.5:1 以上を確保。ライト側は現行値より1〜2段暗く、ダーク側は現行の明るい値をほぼ流用可）。

### 2.2 コンポーネント規約

- `.card` に `padding: var(--card-pad)` をデフォルトで持たせ、余白なしが必要な箇所（画像カード・テーブルカード）だけ `.card.flush` で打ち消す。**`app.js` 内のインライン style は全廃**し、`.card-pad-lg` / `.center` / `.mt-*` 等の最小ユーティリティか専用クラスに置換する。
- ボタン・チップ・セグメント等の既存コンポーネントは class 名を変えない（app.js 側の変更箇所を最小化するため）。見た目の調整はCSSのみで行う。

## 3. 画面別の改修方針

### 3.1 単語カード（最優先。P0-1/2/4 の解消 + リッチ化）

**構造変更（`app.js` の `drawCard()` テンプレート + CSS）:**

1. **絶対配置を全廃**し、表裏それぞれのカードを縦フレックス3段構成にする:
   - ヘッダー行（通常フロー・`display:flex; justify-content:space-between`）: 左にメタチップ群、右に状態バッジ。折り返しても下の本文が押し下がるだけで**構造的に重なりが起きない**。
   - 本文（表: 中央寄せの用語 + 一言定義プレビュー / 裏: 用語 + `dl`）。裏面本文だけを `overflow-y:auto` のスクロール領域にし、ヘッダーは固定表示のまま残す。
   - 裏面スクロール領域の下端に `mask-image` か疑似要素のフェードを敷き、続きがあることを示す（P0-4）。
2. 裏面の `dl` を「定義・説明」/「関連語・混同しやすい語」/「実務例・注意点」でグループ化し、ラベル（`dt`）をチップ風の小ラベルに。特に「混同しやすい語」は amber 系で目立たせる（この試験の肝のため）。
3. デッキ進捗を「n / N」テキストだけでなく細い進捗バー（既存 `.bar` 流用）でも表示。
4. コントロール列の「もう一度」ボタンのインライン style（`app.js:618`）を `.btn.warn` クラス化。
5. フリップアニメーションは既存を維持（reduced-motion 対応済み）。ホバー時にカードへ `--shadow-3` + わずかな lift。

### 3.2 問題演習（P0-3 の解消 + フィードバック強化）

1. `.quiz-top` + `.quiz-progressbar` を1つの **sticky ヘッダー**（`top: 60px`、topbar の直下、背景は topbar と同じ半透明+blur）にまとめる。模試中は常にタイトル・タイマー・進捗が見える。残り5分の `warn` 状態では数字を punch させる（拡大+赤、`prefers-reduced-motion` では色のみ）。
2. 選択肢 `.opt` : 正解に ✓・誤答に ✕ のマークを `key` バッジ側に重ねて表示し、色弱でも判別可能にする（現在の `opt-status` テキストは維持）。
3. フィードバックパネル `.feedback` : 左端に verdict 色のアクセントボーダー（4px）を付け、正解/不正解が一瞥で分かるように。表示時のフェードは既存 keyframe を流用。
4. 演習メニューのカード4種（`quiz-menu-card`）にアイコンの色付き丸背景（quick-grid の `q-ico` と同じ意匠）を導入し、ホームと視覚言語を揃える。
5. カテゴリ別・章別の `cat-btn` はホバーで accent 化する現状仕様を維持しつつ、押下済みカテゴリの成績（正答率チップ）を右側に出せる余地を残す（**実装は任意**。やる場合も STORE の読み取りのみで書き込み構造は変えない）。

### 3.3 結果画面（P2-6）

1. `score-ring` の色を得点帯で変える: ≥90% green / ≥75% teal / ≥60% amber / それ未満 red（`conic-gradient` の色を style 属性で渡している箇所 `app.js:950` を拡張）。
2. リングに `@property` や transition が使えない環境も考慮し、色変更のみで可（アニメーションは任意）。
3. カテゴリ別内訳の `mini-bar` を太く（6px→8px）し、ラベル・分数の整列をグリッド化。

### 3.4 ホーム（P0-5 + P2-2/3）

1. hero を `display:flex`（本文ブロック + カウントダウンブロック）に組み替え、**絶対配置をやめる**。760px 以下は縦積み。`style.css:621` の死にスタイルを削除。
2. hero の背景に、単色グラデに重ねる控えめな装飾（大きな半透明円 or `radial-gradient` のメッシュ2〜3個、CSSのみ）を追加して平板さを解消。文字コントラストは `#fff` on indigo を維持。
3. 統計カード4枚: 数値を `--fs-2xl`、バーを 7px→8px にし、達成率100%のとき green 系に切り替わる小演出。
4. 「今日のおすすめ」`next-card` を accent の左ボーダー付きカードにして、ページ内で最初に目が行く見た目にする（学習系サイトの「次の一歩」導線として最重要のため）。

### 3.5 読む（P2-1）

1. 章リスト `ch-item` に **`site/assets/read-visuals/ch0N-*.png` のサムネイル**（横 96〜120px、`aspect-ratio` 維持、`border-radius: var(--radius-sm)`、`loading="lazy"`）を追加する。章番号バッジは画像左上に小さく重ねるか画像の左に維持。`CHAPTER_VISUALS`（`app.js:334-398`）に src があるので `viewReadList()` から参照できる。9章すべて画像が存在することを確認済み。
2. 読了章は画像に緑のチェックオーバーレイ + 彩度を落とす等、状態が一覧で分かる表現に。
3. 章ページ: `chapter-visual` の図はそのまま、`visual-note`（読み方・誤解ポイント）の2項目をアイコン付き2カラム（モバイル1カラム）に整理。
4. TOC カードに現在地ハイライト（`IntersectionObserver` で現在セクションの リンクに `.current` を付ける。**実装は任意**、20行程度で済む場合のみ）。

### 3.6 検索・用語一覧・計画（P1-5/8）

1. 用語一覧テーブル: 760px 以下ではテーブルをカード型リスト（用語+チップ+定義の縦積み）に切り替える。CSS のみで実現（`thead` 非表示 + `td` block化 + `data-label` 属性を `app.js:653-660` で付与）。
2. 検索ヒットカードにヒット種別のアイコン/チップ（語句・本編・問題）を付け、ホバーの lift を quick-grid と統一。
3. 計画タブ `.tabs` : アクティブタブの下線を 2px→3px にし、`doc-render` のテーブルに `prose-table` と同じゼブラ・角丸スタイルを適用（現在2種類のテーブル様式が併存しているのを統一）。

### 3.7 全体・シェル（P1-6/7 + P2-5）

1. `index.html`: 未使用 preconnect 削除、`<meta name="theme-color">`（ライト/ダーク両対応の media 指定）、インライン SVG favicon（`data:image/svg+xml` で「UX」ブランドマーク相当）を追加。
2. topbar: アクティブナビの下に小さなインジケータ、スクロール時のみ border+shadow を強調（`scroll` リスナー1つ。任意）。
3. マイクロインタラクション最小セット: `.btn:active` の scale(0.98)、カード hover lift の統一（`translateY(-2px)` + `--shadow-3`）、ページ描画時の `main` フェードイン（`setView` で class 付け直し、150ms）。**すべて `prefers-reduced-motion: reduce` で無効化されることを確認する。**
4. フッターを2行の免責文＋リンク行に整理し、上マージンをトークン化。

## 4. 実装フェーズと受け入れ基準

依存関係: A → B → C の順。C 内の項目は独立なので任意の順で可。

### Phase A: デザイントークン整備 + 重なり根絶（P0 全件）

- §2 のトークン導入、既存値のトークン置換（見た目の変化は最小限）
- §3.1-1（フラッシュカードのフレックス化）、§3.2-1（クイズ sticky ヘッダー）、§3.4-1（hero フレックス化）
- `app.js` 内インライン style の全廃（P1-1）

**受け入れ基準:**
- 375px / 768px / 1280px の3幅 × ライト/ダークで、単語カード（チップが最多の語=重要度A+長いカテゴリ名で確認）・hero・クイズ実行画面に要素の重なりがない
- 模試モードで画面下部までスクロールしてもタイマーと進捗バーが見えている
- 単語カード裏面で最終フィールド「試験での注意点」までスクロールで到達でき、続きがあることが視覚的に分かる
- `grep -n 'style="' site/js/app.js` のヒットが 動的値（progress width, score-ring の --p 等、値が実行時に決まるもの）のみになっている
- `node site/build.js` exit 0

### Phase B: 一貫性・可読性（P1 残件）

- チップ文字色のコントラスト修正（P1-4）、タイポスケール適用（P1-2/3）
- 用語一覧テーブルのモバイルカード化（P1-5）、index.html の掃除と meta 追加（P1-6/7）
- 検索ヒット・タブ・doc-render テーブルの統一（P1-8, §3.6）

**受け入れ基準:**
- チップ全色（accent/teal/green/amber）の文字/背景コントラストがライト・ダーク双方で 4.5:1 以上（実測値をワークログに記録）
- 375px 幅で用語一覧が横スクロールなしで読める
- ブラウザタブに favicon が出る。未使用 preconnect が消えている

### Phase C: リッチ化（P2 全件）

- 章リストのサムネイル化（P2-1）、hero 装飾（P2-2）、進捗・達成演出（P2-3）
- クイズフィードバック強化（P2-4, §3.2-2/3）、スコアリング色分け（P2-6）
- マイクロインタラクション（P2-5, §3.7-3）

**受け入れ基準:**

## 5. 動作確認の方法
