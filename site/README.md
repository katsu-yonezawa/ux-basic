# UX検定基礎 学習サイト

`docs/` 配下の教材（試験対策ガイド＋重要語句集）を、ブラウザで「読む・覚える・解く」ことができる学習サイトです。
ビルド不要・依存なしの静的サイトで、`site/index.html` をブラウザで開くだけで動作します。

## 使い方

`site/index.html` をダブルクリックして開くだけです（サーバー不要）。
学習の進捗（読了章・単語の習得状況・問題の成績）は、ブラウザの localStorage に自動保存されます。

### 主な機能

| 機能 | 内容 |
|---|---|
| **ホーム** | 試験までのカウントダウン、学習進捗サマリー、試験概要 |
| **読む** | 本編テキスト全9章。章内目次・読了管理・章末ミニ問題への導線 |
| **単語カード** | 重要語句110語以上の暗記カード。重要度・カテゴリ表示、めくり／「覚えた・要復習」の仕分け／フィルタ／シャッフル（キーボード操作可） |
| **問題演習** | 本番モード模試（100問100分・採点）、即時解説の演習、カテゴリ別／章別、**間違い復習**、カテゴリ別の正答内訳 |
| **検索** | 重要語句・本編・問題を横断検索 |
| **計画・資料** | 試験概要、シラバス網羅表、直前復習メモ、2週間／4週間学習計画 |

ライト／ダークテーマ切替（右上の ◐）、スマホ対応。

## 教材を更新したら

学習コンテンツは `docs/*.md` と `docs/syllabus_keywords.json` から自動生成しています。Markdown やシラバス語句管理ファイルを編集したら、次を実行してサイトに反映します。

```bash
node site/build.js
```

これで `site/js/data.js`（全コンテンツのデータ）が再生成されます。
`site/js/data.js` は生成物です。直接編集せず、必ず元データを直してから `node site/build.js` を実行してください。

ビルド時には、次の検査を行います。

- 章数9、各章ミニ問題5問、模擬問題100問、用語集110語以上
- 章セクション、問題文、4択、正解、解説、不正解理由、重複ID、重複問題文
- 模擬問題と章末問題の正解位置分布
- 弱い誤答語の簡易チェック
- `docs/syllabus_keywords.json` の構造、重複語句、A/B優先語句の教材内対応

エラーがある場合は終了コードが1になり、GitHub Actionsなどでも検知できます。

## ファイル構成

```
site/
├── index.html      … サイトの土台（ヘッダ・ナビ・フッタ）
├── css/style.css   … スタイル（ライト/ダーク・レスポンシブ）
├── js/
│   ├── data.js     … docs から自動生成したデータ（直接編集しない）
│   └── app.js      … アプリ本体（ルーター・各画面・クイズエンジン）
└── build.js        … docs/*.md → js/data.js の変換スクリプト
```

## シラバス語句管理

`docs/syllabus_keywords.json` は、公式シラバス由来の大カテゴリ・小カテゴリ・重要語句・優先度・対応章を管理するファイルです。A優先語句は、本文、用語集、問題文、解説のいずれかに出ていない場合、ビルドエラーになります。B優先語句は本文または用語集に出ていない場合に警告します。

公式情報を更新するときは、次の順で確認してください。

1. 公式サイト、公式シラバスPDF、該当回の公式案内を確認する。
2. `docs/ux_kentei_basic_hcd_study_guide.md` の確認日、試験概要、シラバス表を更新する。
3. 必要に応じて `docs/syllabus_keywords.json` と `docs/ux_kentei_basic_glossary.md` を更新する。
4. 問題の正解位置分布が偏らないよう確認する。
5. `node site/build.js` を実行し、`site/js/data.js` を再生成する。

## 注意

本サイトの模擬問題はすべてオリジナルであり、公式問題ではありません。合格基準・出題割合は公式に断定されていません。受験前に必ず公式サイトと最新シラバスを確認してください。
