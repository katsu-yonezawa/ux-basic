# UX検定基礎（HCD検®認定）重要語句集

作成日: 2026-06-13  
出典: 「UX検定基礎（HCD検®認定）試験対策教材」（ux_kentei_basic_hcd_study_guide.md）から分離

この語句集は暗記カードとして使えますが、必ず各語句の「実務例」と「試験での注意点」まで読んでください。

## UX

- 用語: UX
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: ユーザーが製品・サービスと関わることで得る体験全体
- 詳しい説明: 利用前の期待、利用中の操作、利用後の印象や価値認識まで含む。UIやユーザビリティだけに限定しない。
- 関連語: UI、CX、ユーザビリティ
- 混同しやすい語: UI
- 実務例: 予約アプリで検索、予約、通知、来店受付までが一連の体験になる。
- 試験での注意点: UXを「UI」「画面デザイン」と同一視する選択肢は誤り。利用前の期待から利用後の印象まで含む体験全体を指す。

## UI

- 用語: UI
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: ユーザーがシステムとやり取りする接点
- 詳しい説明: ボタン、フォーム、メニュー、画面レイアウトなど、操作や表示に関わる部分。
- 関連語: UX、情報設計、インタラクション
- 混同しやすい語: UX
- 実務例: 社内申請システムの入力フォームや承認ボタン。
- 試験での注意点: UIはUXを構成するが、UXそのものではない。

## UXインテリジェンス

- 用語: UXインテリジェンス
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: ユーザーに向き合い価値提供するための共通能力
- 詳しい説明: デジタル前提の社会で、専門職だけでなく多様な職種が持つべきUXの知識、姿勢、実践力。
- 関連語: UX、UXグロース、UX組織化
- 混同しやすい語: UXデザイナーの専門技能
- 実務例: エンジニアが仕様の背後にある利用状況を確認する。
- 試験での注意点: 特定職種だけの能力とする説明は狭い。

## UXグロース

- 用語: UXグロース
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: 体験価値を継続的に改善・成長させる活動
- 詳しい説明: リリース後の行動データ、調査、運用知見をもとにユーザー価値と事業価値を高める。
- 関連語: グロースチーム、UX運用、DevOps
- 混同しやすい語: 広告運用のみ
- 実務例: 問い合わせ分析からヘルプ導線を改善し継続利用を高める。
- 試験での注意点: 短期KPIだけに限定しない。

## OMO

- 用語: OMO
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: オンラインとオフラインを統合して体験を設計する考え方
- 詳しい説明: アプリ、店舗、サポートなどの接点を分断せず、連続した利用文脈として扱う。
- 関連語: DX、CX、行動データ
- 混同しやすい語: O2O
- 実務例: アプリで予約し、店舗でスムーズに受付する。
- 試験での注意点: 混同語のO2Oが「オンラインからオフラインへの送客」を主眼にするのに対し、OMOは「オンラインとオフラインの融合」を指す。オンライン施策だけを指す選択肢は誤り。

## D2C

- 用語: D2C
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: 企業が中間流通を介さず顧客と直接関係を持つモデル
- 詳しい説明: 販売だけでなく、顧客データや継続的な関係を活かした体験改善と関わる。
- 関連語: CX、データ活用、ブランド
- 混同しやすい語: ECサイト一般
- 実務例: メーカーが自社アプリで購入後サポートまで提供する。
- 試験での注意点: 直販形式だけでなく顧客関係の設計が重要。

## アフターデジタル

- 用語: アフターデジタル
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: デジタルが生活に浸透した状態を前提にした考え方
- 詳しい説明: オンラインが一部ではなく、リアルの行動もデジタル接点と接続される環境を想定する。
- 関連語: OMO、行動データ、UXグロース
- 混同しやすい語: 単なるIT化
- 実務例: 店舗行動とアプリ利用をつなげて体験を設計する。
- 試験での注意点: デジタル画面だけの話ではない。

## UXハニカム

- 用語: UXハニカム
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: 有用性・使いやすさ・望ましさ・見つけやすさ・信頼性・アクセス可能性・価値の7要素でUXを評価するモデル
- 詳しい説明: モービルが提唱した7要素。有用性、使いやすさ、望ましさ、見つけやすさ、信頼性、アクセス可能性、価値の各面からUXを多面的に評価する。
- 関連語: ユーザビリティ、アクセシビリティ
- 混同しやすい語: ビジュアル評価表
- 実務例: ECで商品が探しやすく、信頼でき、購入後も満足できる状態。
- 試験での注意点: 使いやすさだけでUX全体を判断しない。

## UXの5階層モデル

- 用語: UXの5階層モデル
- 重要度: A
- カテゴリ: UXインテリジェンスの理念
- 一言定義: UXを戦略から表層まで段階的に捉えるモデル
- 詳しい説明: 戦略、要件、構造、骨格、表層のように、目的から画面表現までを接続して考える。
- 関連語: 情報設計、UI、要求定義
- 混同しやすい語: 画面デザイン工程
- 実務例: 管理画面の目的、機能、情報構造、ワイヤー、見た目を順に整理する。
- 試験での注意点: 表層だけをUXと考えない。

## ユーザビリティ

- 用語: ユーザビリティ
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 特定状況で目標を有効・効率・満足に達成できる程度
- 詳しい説明: 目標達成、時間や手間、満足度を含む利用品質。UXの重要要素。
- 関連語: ISO 9241-11、UX、アクセシビリティ
- 混同しやすい語: UX
- 実務例: 予約変更を迷わず短時間で完了できる。
- 試験での注意点: 「使いやすさ全般」とだけ捉える選択肢は不十分。ISO 9241-11では特定のユーザー・利用状況・目標を前提に、有効さ・効率・満足で測る。

## アクセシビリティ

- 用語: アクセシビリティ
- 重要度: A
- カテゴリ: UX関連基礎知識
- 一言定義: 多様な利用者が情報や機能を利用できる状態
- 詳しい説明: 障害、年齢、環境、端末、状況の違いに配慮し、利用可能性を高める。
- 関連語: WCAG、ユニバーサルデザイン
- 混同しやすい語: ユーザビリティ
- 実務例: キーボード操作、代替テキスト、十分なコントラスト。
- 試験での注意点: 「障害者向けの特別対応」に限定する選択肢は誤り。年齢・環境・状況の多様性に配慮し、誰もが情報や機能を利用できる状態を指す。

## ユニバーサルデザイン

- 用語: ユニバーサルデザイン
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: できるだけ多くの人が使えるように設計する考え方
- 詳しい説明: 年齢や能力にかかわらず利用しやすい設計を目指す。
- 関連語: アクセシビリティ、人間工学
- 混同しやすい語: バリアフリー
- 実務例: 入力フォームで分かりやすいラベルとエラー説明を用意する。
- 試験での注意点: アクセシビリティと重なるが同義ではない。

## デザイン思考

- 用語: デザイン思考
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 人間中心で課題探索と創造的解決を行う考え方
- 詳しい説明: 共感、問題定義、発想、試作、検証を通じて曖昧な課題に向き合う。
- 関連語: HCD、ダブルダイヤモンド
- 混同しやすい語: 美術制作
- 実務例: 業務課題を観察し、複数案を試作して検証する。
- 試験での注意点: 装飾や感性だけの話ではない。

## ダブルダイヤモンド

- 用語: ダブルダイヤモンド
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 発散と収束を二度行う問題解決モデル
- 詳しい説明: 課題発見で発散・収束し、解決策で発散・収束する流れを示す。
- 関連語: デザイン思考、アイデア創出
- 混同しやすい語: ウォーターフォール
- 実務例: 利用者課題を広く探り、焦点化してから解決案を広げて選ぶ。
- 試験での注意点: 最初から解決策に飛びつかない。

## アジャイル

- 用語: アジャイル
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 変化に対応しながら反復的に価値を届ける開発の考え方
- 詳しい説明: 短いサイクルで計画、実装、レビュー、改善を行う。
- 関連語: スクラム、スプリント、バックログ
- 混同しやすい語: HCD
- 実務例: スプリントレビューでユーザー観点のフィードバックを得る。
- 試験での注意点: 「アジャイルで開発すれば自動的にUXが良くなる」とする選択肢は誤り。反復にユーザー評価を組み込むことが必要。

## スクラム

- 用語: スクラム
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 役割・イベント・作成物で構成されるアジャイル開発の代表的フレームワーク
- 詳しい説明: スプリント、プロダクトバックログ、スクラムイベント、役割（プロダクトオーナー・スクラムマスター・開発者）などで構成される。
- 関連語: アジャイル、スプリント、インクリメント、スクラムマスター
- 混同しやすい語: XP
- 実務例: 2週間スプリントで改善機能を提供する。
- 試験での注意点: 役割とイベントの「枠組み」を定める。TDDやペアプログラミングなど技術プラクティス中心のXPとは対象が異なる。

## バックログ

- 用語: バックログ
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 実現したい項目を優先順位付きで管理する一覧
- 詳しい説明: 機能、改善、調査、技術課題などを整理し、価値に基づいて順序づける。
- 関連語: スクラム、ユーザーストーリー
- 混同しやすい語: WBS
- 実務例: UX改善の仮説をバックログに入れる。
- 試験での注意点: 単なる要望箱にしない。

## リーンUX

- 用語: リーンUX
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 仮説検証と学習を重視するUX実践
- 詳しい説明: 詳細な成果物より、チームの学習と小さな検証を重視する。
- 関連語: リーンスタートアップ、MVP
- 混同しやすい語: アジャイル
- 実務例: 新機能案をプロトタイプで検証してから実装する。
- 試験での注意点: ドキュメント不要論ではない。

## MVP

- 用語: MVP
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 重要仮説を学ぶための最小限の提供物
- 詳しい説明: 完成品ではなく、価値や需要を検証するために必要最小限で作る。
- 関連語: リーンUX、仮説検証
- 混同しやすい語: 低品質版
- 実務例: AI回答機能を人手運用で先に試す。
- 試験での注意点: 雑なものを出すことではない。

## パーパス

- 用語: パーパス
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 組織の存在意義
- 詳しい説明: なぜその組織が存在するのかを示し、UXや事業判断の上位方針になる。
- 関連語: ミッション、ビジョン、バリュー
- 混同しやすい語: スローガン
- 実務例: プロダクト改善で提供価値の判断基準にする。
- 試験での注意点: 見栄えのよい言葉ではなく意思決定に使う。

## 行動経済学

- 用語: 行動経済学
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 人間の非合理な意思決定を扱う学問領域
- 詳しい説明: プロスペクト理論、ナッジ、アンカリングなどを通じて選択行動を理解する。
- 関連語: 認知バイアス、ナッジ
- 混同しやすい語: 心理操作
- 実務例: 初期値や選択肢提示をユーザー利益に沿って設計する。
- 試験での注意点: 不当誘導と混同しない。

## ナッジ

- 用語: ナッジ
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 望ましい選択を自然に支援する設計
- 詳しい説明: 選択の自由を残しながら、ユーザーや社会に望ましい行動を促す。
- 関連語: 行動経済学、選択設計
- 混同しやすい語: ダークパターン
- 実務例: 入力漏れが起きやすい項目に分かりやすい例を出す。
- 試験での注意点: ユーザーの不利益を隠す設計は不適切。

## 認知バイアス

- 用語: 認知バイアス
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 判断や認識に生じる偏り
- 詳しい説明: 人間は情報を完全に合理的に処理できないため、設計上の配慮が必要になる。
- 関連語: 行動経済学、認知心理学
- 混同しやすい語: 個人の能力不足
- 実務例: 無料トライアルの終了条件を誤解しやすい表示を避ける。
- 試験での注意点: バイアスを悪用しない。

## 認知負荷

- 用語: 認知負荷
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 情報処理にかかる心理的負担
- 詳しい説明: 項目が多すぎる、ラベルが分かりにくい、記憶を要求するなどで増える。
- 関連語: 認知心理学、情報設計
- 混同しやすい語: サーバー負荷
- 実務例: 複雑な設定をステップ分割する。
- 試験での注意点: 情報量を減らすだけでなく構造化が重要。

## ヤコブの法則

- 用語: ヤコブの法則
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: ユーザーは他サイトでの経験に基づいて期待するという考え方
- 詳しい説明: 慣れたパターンを活かすことで学習負荷を下げられる。
- 関連語: 認知心理学、UIパターン
- 混同しやすい語: ヤコブ・ニールセンのユーザビリティ定義
- 実務例: ECのカート位置や検索欄の配置を慣習に合わせる。
- 試験での注意点: 独自性が常に良いとは限らない。

## ヒックの法則

- 用語: ヒックの法則
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 選択肢が増えるほど判断時間が増えるという考え方
- 詳しい説明: 選択肢を整理し、段階的に提示する設計に役立つ。
- 関連語: 認知負荷、情報設計
- 混同しやすい語: フィッツの法則
- 実務例: 料金プランを比較しやすくまとめる。
- 試験での注意点: 選択肢をただ減らせばよいわけではない。

## 文化人類学

- 一言定義: 人々の生活文化や文脈を理解する学問領域
- 詳しい説明: 観察やフィールドワークを通じ、ユーザーの行動を文化的背景から理解する。

- 実務例: 現場の暗黙ルールを観察して業務改善する。

## 人間工学

- 用語: 人間工学
- 重要度: B
- カテゴリ: UX関連基礎知識
- 一言定義: 人間の身体・知覚特性に合う設計を扱う領域
- 詳しい説明: 身体寸法、視認性、操作性、作業負荷などを考慮する。

## プロジェクトマネジメント

- 用語: プロジェクトマネジメント
- 重要度: B
- カテゴリ: UXプロジェクト計画
- 一言定義: 期限やスコープを持つ取り組みを管理する活動
